Design Principles

The design of the OPeNDAP software is based on the following two considerations:

  1. Data are often most appropriately distributed by the individual or group that has developed them.
  2. The user will in general like to access data from the application software with which they are most familiar.

This has resulted in a highly distributed system that allows users to control the distribution of their own data and the way they access data from remote sites.

User Application arrow_forward DAP Client

Once a user's data analysis application (or API) is made network-savvy with the OPeNDAP tools, the scope of an application's search for data is extended, because remote data becomes as accessible as local data. A DAP-enabled application can...

  • Get any data anywhere on the web that is served by a DAP server.
  • Use data from any DAP server, regardless of its native format.
  • Still perform all its original functions for accessing data locally.

The truly remarkable part: you don't have to rewrite the code to enable many applications to work as a DAP clients.

Not Just Oceanography

Although the OPeNDAP software was originally designed and developed by oceanographers and computer scientists for oceanographic data (as part of the DODS and NVODS projects), there is nothing in the design of the OPeNDAP software that constrains its use to oceanography.

Indeed, it has been adopted by the High Altitude Observatory community and is being considered by segments of the meteorological and space science communities.

Community-Driven Projects

OPeNDAP software is open-source and is intended to facilitate building community-driven projects for developing end-to-end systems that can locate, understand, access, and use scientific data. OPeNDAP has already developed a flexible data model, a transport protocol, and several software frameworks, including...

  • C++ and Java implementations of the data model and transport protocol
  • A set of DODS servers
  • A set of DODS clients

What Does OPeNDAP Do For You?

OPeNDAP allows you to access remote data over the web...

  • Using familiar data analysis and visualization packages, such as Matlab, Ferret, and IDL.
  • Without worrying about data storage formats.
  • Accessing only the data of interest.

OPeNDAP also allows you to make your data available remotely. Your data can be in a variety of formats, including netCDF, HDF, and many more.

Our Software

Hyrax The next generation server from OPeNDAP Hyrax uses a modular design that employs a lightweight Java servlet to provide the public-accessible client interface, and a back-end daemon, the BES, to handle the heavy lifting. The BES uses the same handlers that are used with Server3 (also know as the CGI Server) but loads those at run time. Libdap C++ SDK that contains an implementation of DAP 2.0 and DAP4.0 The libdap includes both Client- and Server-side support classes. Our software distributions are signed using a public/private key pair.
